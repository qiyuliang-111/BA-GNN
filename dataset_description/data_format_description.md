# Graph Data Format Description

This document describes the graph-structured dataset generated by
`dataset_creation_code/dataset_generation.py`.

## 1. Sample Definition

Each `.pt` file corresponds to one airfoil geometry at one angle of attack.

The output filename follows:

```text
{airfoil}_aoa_{angle_of_attack:.1f}.pt
```

Example:

```text
NACA0012_aoa_4.0.pt
```

Each file stores one `torch_geometric.data.Data` object.

## 2. Graph Object

For a graph with `N` nodes and `E` unique mesh edges, the saved object contains
the following fields.

| Field | Shape | Data type | Description |
|---|---:|---|---|
| `x` | `[N, 4]` | `torch.float32` | Node-feature matrix: `[x, y, node_type, angle_of_attack]` |
| `edge_index` | `[2, E]` | `torch.long` | Mesh connectivity constructed from unique edges |
| `edge_attr` | `[E, 5]` | `torch.float32` | Geometric edge features |
| `y` | `[N, 3]` | `torch.float32` | Target flow-field variables: `[Cp, U, V]` |
| `node_type` | `[N]` | `torch.long` | Integer node-type labels |
| `edge_type` | `[E]` | `torch.long` | Integer edge-type labels |
| `matched_indices` | `[M]` | NumPy array | Mesh-node indices matched to the source flow-field records |
| `matching_distances` | `[M]` | NumPy array | Nearest-neighbor matching distances |

Here, `M` is the number of flow-field records read from the corresponding ASCII
file. In the released dataset, the flow-field records are matched to mesh nodes
using a KD-tree nearest-neighbor search.

## 3. Node Features

The node-feature matrix is defined as

```text
x = [x_coordinate, y_coordinate, node_type, angle_of_attack]
```

The four components are:

| Index | Feature | Description | Unit |
|---:|---|---|---|
| 0 | `x_coordinate` | Cartesian x-coordinate read directly from the CGNS mesh | Same as the source mesh |
| 1 | `y_coordinate` | Cartesian y-coordinate read directly from the CGNS mesh | Same as the source mesh |
| 2 | `node_type` | Numerical encoding of the node category | Dimensionless |
| 3 | `angle_of_attack` | Angle of attack assigned to all nodes in the graph | Degree |

The spatial coordinates are not normalized during dataset construction.

## 4. Node-Type Encoding

The node categories are encoded as:

| Value | Node type |
|---:|---|
| 0 | Interior-fluid node |
| 1 | Airfoil-boundary node |
| 2 | Far-field-boundary node |

The same labels are also stored separately in the `node_type` tensor.

## 5. Graph Connectivity

The mesh connectivity is constructed from triangular and quadrilateral cells
read from the CGNS file.

For every cell, its boundary edges are collected. Duplicate occurrences are
counted, and each unique edge is stored once in `edge_index`.

The endpoint order of each stored edge is canonicalized as:

```text
(min(node_i, node_j), max(node_i, node_j))
```

Therefore, `edge_index` contains the unique mesh-edge list generated by the
dataset-construction script.

## 6. Edge Features

For each edge from source node `i` to destination node `j`, the geometric edge
feature is

```text
edge_attr = [
    delta_x_normalized,
    delta_y_normalized,
    edge_length_normalized,
    unit_direction_x,
    unit_direction_y
]
```

The components are calculated as follows:

```text
delta = coordinate_j - coordinate_i
length = ||delta||
delta_normalized = delta / maximum_edge_length
edge_length_normalized = length / maximum_edge_length
unit_direction = delta / (length + 1e-12)
```

The maximum edge length is calculated separately for each airfoil mesh.

All five edge features are dimensionless.

**Note:** the edge category is not included in `edge_attr`; it is stored
separately in the `edge_type` tensor.

## 7. Edge-Type Encoding

The edge categories are encoded as:

| Value | Edge type |
|---:|---|
| 0 | Internal mesh edge |
| 1 | Airfoil-boundary edge |
| 2 | Far-field-boundary edge |

An edge shared by more than one cell is classified as an internal edge. A
single-occurrence edge is classified as an airfoil-boundary or far-field edge
according to the automatic boundary-recognition rule used in the construction
script.

## 8. Target Variables

The target matrix is defined as

```text
y = [Cp, U, V]
```

| Index | Variable | Description | Unit |
|---:|---|---|---|
| 0 | `Cp` | Pressure coefficient | Dimensionless |
| 1 | `U` | x-direction velocity component | m/s |
| 2 | `V` | y-direction velocity component | m/s |

The target variables are stored in physical space. Z-score standardization is
performed later during model training and is not applied by the
dataset-construction script.

## 9. Flow-Field-to-Mesh Matching

The source ASCII flow-field file is expected to contain at least six columns:

```text
id, x, y, Cp, U, V
```

The flow-field coordinates are matched to CGNS mesh nodes using
`scipy.spatial.cKDTree`.

The matching tolerance used in the dataset-construction script is:

```text
1e-4
```

The script stores both the matched mesh-node indices and the corresponding
nearest-neighbor distances in each graph object.

## 10. Boundary Recognition

Boundary recognition is performed automatically from mesh topology and node
coordinates.

The procedure consists of:

1. constructing unique mesh edges and counting their cell occurrences;
2. identifying single-occurrence boundary edges;
3. distinguishing airfoil and far-field edges using the far-field threshold;
4. assigning node types from the classified boundary edges.

The far-field threshold used in the released construction script is:

```text
0.78
```

## 11. Input Directory Convention

The construction script expects the following directory structure:

```text
dataset/
└── AIRFOIL_NAME/
    ├── CGNS/
    │   └── mesh_file.cgns
    ├── ASCII/
    │   └── flow_field_files
    └── dataset_pyg/
        └── generated_graph_files.pt
```

For data-processing convenience, some internal airfoil directory names include
the prefix `NACA`, such as `NACAGOE`. This is only a filename convention and
does not indicate the actual airfoil family.

## 12. Loading Example

```python
import torch

data = torch.load(
    "NACA0012/dataset_pyg/NACA0012_aoa_4.0.pt",
    map_location="cpu",
)

print(data)
print(data.x.shape)
print(data.edge_index.shape)
print(data.edge_attr.shape)
print(data.y.shape)
print(data.node_type.shape)
print(data.edge_type.shape)
```
